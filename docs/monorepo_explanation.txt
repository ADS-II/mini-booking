### Descripción del Monorepo "Mini-Booking"

Este proyecto está organizado como un "monorepo", lo que significa que todo el código fuente y la configuración de la aplicación "Mini-Booking" se encuentran en este único repositorio.

El objetivo de esta estructura es centralizar todos los componentes del sistema para facilitar el desarrollo, la gestión y el despliegue de la aplicación de manera cohesiva.

Las responsabilidades de cada directorio principal son las siguientes:

*   `backend/`
    Contiene el código fuente de la API RESTful. Está construido con Java y el framework Spring Boot, y se encarga de toda la lógica de negocio, el acceso a los datos y la comunicación con la base de datos PostgreSQL.

*   `frontend/`
    Contiene el código fuente de la interfaz de usuario. Es una Aplicación de Una Sola Página (SPA) construida con Angular, con la que los usuarios interactúan directamente desde su navegador.

*   `infra/`
    Define la infraestructura de la aplicación como código. Incluye el archivo `docker-compose.yml` para orquestar los contenedores de desarrollo y los `Dockerfile` para construir las imágenes del frontend y el backend.

*   `docs/`
    Es el directorio central para toda la documentación del proyecto, incluyendo los diagramas de arquitectura, explicaciones de conceptos y cualquier otro material de referencia.

*   `.github/`
    Almacena las configuraciones para la Integración Continua y el Despliegue Continuo (CI/CD) utilizando GitHub Actions. Automatiza tareas como la construcción y prueba del código cada vez que se realiza un cambio.

Este enfoque nos permite realizar cambios que afectan a varios componentes (por ejemplo, al backend y al frontend a la vez) en una sola confirmación (commit), asegurando que todas las partes del sistema evolucionen de manera sincronizada.
